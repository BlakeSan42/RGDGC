STEP 1: GENERATE SECRETS (5 minutes)
================================================================================

Run these commands and save the output somewhere safe (password manager):

  cd /Users/blakesanders/RGDGC

  # Generate SECRET_KEY
  python3 -c "import secrets; print('SECRET_KEY=' + secrets.token_hex(32))"

  # Generate JWT_SECRET
  python3 -c "import secrets; print('JWT_SECRET=' + secrets.token_hex(32))"

  # Generate OWNER_KEY (your super-secret admin override key)
  python3 -c "import secrets; print('OWNER_KEY=' + secrets.token_hex(32))"

Put all three in your .env file:

  cp .env.example .env
  # Edit .env and paste the three values above

STEP 2: DEPLOY BACKEND TO RAILWAY (15 minutes)
================================================================================

A. Create Railway account
   1. Go to https://railway.app
   2. Sign up with GitHub
   3. Click "New Project" → "Deploy from GitHub Repo"
   4. Select the RGDGC repo
   5. Railway auto-detects the backend directory

B. Add PostgreSQL
   1. In your Railway project → "New" → "Database" → "PostgreSQL"
   2. Wait for it to provision (~30 seconds)
   3. Click the database → "Variables" tab → copy DATABASE_URL
   4. It looks like: postgresql://postgres:xxx@xxx.railway.internal:5432/railway

C. Add Redis
   1. "New" → "Database" → "Redis"
   2. Copy REDIS_URL from its variables tab

D. Configure backend service
   1. Click your backend service → "Variables" tab
   2. Add these variables (click "Raw Editor" for bulk paste):

      DATABASE_URL=<paste from step B, change to +asyncpg>
      REDIS_URL=<paste from step C>
      SECRET_KEY=<from Step 1>
      JWT_SECRET=<from Step 1>
      OWNER_KEY=<from Step 1>
      ENVIRONMENT=production
      CORS_ORIGINS=https://admin.rgdgc.com,https://rgdgc.com,rgdgc://

   3. Set the root directory to "backend" in service settings
   4. Set start command: uvicorn app.main:app --host 0.0.0.0 --port $PORT

E. Deploy
   1. Railway auto-deploys on push to main
   2. Or click "Deploy" manually
   3. Wait for build to complete (~2-3 minutes)
   4. Click the generated URL → add /health to verify
   5. You should see: {"status":"healthy","service":"rgdgc-api"}

F. Run migrations
   1. In Railway → backend service → "Settings" → "Execute Command"
   2. Run: alembic upgrade head
   3. Then run: python scripts/seed_data.py

G. Note your backend URL
   Save it — you'll need it for the mobile app and admin dashboard.
   It looks like: https://rgdgc-backend-production.up.railway.app

STEP 4: SET UP EXPO & BUILD MOBILE APP (20 minutes)
================================================================================

A. Install EAS CLI (if not done)
   npm install -g eas-cli

B. Log in to Expo
   eas login
   # Create account at https://expo.dev if needed

C. Initialize project
   cd /Users/blakesanders/RGDGC/mobile
   eas init
   # Save the Project ID it gives you

D. Update app.json with project ID
   Edit mobile/app.json → extra → eas → projectId
   Paste the UUID from step C

E. Create .env for production
   cat > .env << 'EOF'
   EXPO_PUBLIC_API_URL=<your Railway backend URL>
   EXPO_PUBLIC_EAS_PROJECT_ID=<from step C>
   EXPO_PUBLIC_MAPBOX_TOKEN=<from Mapbox, see CREDENTIALS_SETUP.txt>
   EOF

F. Configure build profiles
   eas build:configure
   # Creates eas.json — defaults are fine

G. Test with Expo Go first (fastest)
   npx expo start
   # Scan QR code with Expo Go on your iPhone
   # Verify: register, login, browse courses, start a round

H. Build for TestFlight (iOS)
   eas build --platform ios --profile preview
   # This takes ~15 minutes
   # When done, it gives you a link to install via TestFlight

I. Build for Android internal testing
   eas build --platform android --profile preview
   # Download the APK and install on your Android device

J. Production builds (when ready for App Store)
   eas build --platform ios --profile production
   eas submit --platform ios
   # Requires Apple Developer account ($99/year)

   eas build --platform android --profile production
   eas submit --platform android
   # Requires Google Play Console ($25 one-time)

STEP 5: SET UP STRIPE (10 minutes, optional for launch)
================================================================================

You can launch without Stripe. Players can check in for free events.
Add Stripe later when you want to charge league fees.

A. Create Stripe account
   1. Go to https://dashboard.stripe.com/register
   2. Complete business verification

B. Get test keys
   1. Dashboard → Developers → API Keys
   2. Copy "Publishable key" (pk_test_...)
   3. Copy "Secret key" (sk_test_...)

C. Set up webhook
   1. Developers → Webhooks → "Add endpoint"
   2. URL: <your Railway backend URL>/api/v1/payments/webhook
   3. Events to listen for: checkout.session.completed
   4. Copy the "Signing secret" (whsec_...)

D. Add to Railway env vars
   STRIPE_SECRET_KEY=sk_test_...
   STRIPE_PUBLISHABLE_KEY=pk_test_...
   STRIPE_WEBHOOK_SECRET=whsec_...

   1. Go to https://console.anthropic.com
   2. API Keys → Create Key
   3. Copy the key (sk-ant-...)

D. Deploy the bot
   Add to Railway as a separate service:
   - Root directory: openclaw-skills
   - Start command: python main.py
   - Environment variables:
     RGDGC_API_URL=<Railway backend URL>
     DISCORD_BOT_TOKEN=<from step A>
     ANTHROPIC_API_KEY=<from step C>

STEP 8: PROMOTE YOURSELF TO SUPER_ADMIN (2 minutes)
================================================================================

After the backend is deployed and seeded:

  # Option A: Via the API (use Swagger docs)
  1. Open <backend URL>/docs
  2. Register your real account
  3. Connect to the Railway database:
     railway connect postgres
  4. Run: UPDATE users SET role = 'super_admin' WHERE email = 'your@email.com';

  # Option B: Via the owner endpoint (if you set OWNER_KEY)
  1. Login to get your JWT token
  2. curl -X POST <backend URL>/api/v1/owner/override-role \
       -H "Authorization: Bearer <your token>" \
       -H "X-Owner-Key: <your OWNER_KEY>" \
       -H "Content-Type: application/json" \
       -d '{"user_id": <your user id>, "role": "super_admin"}'

================================================================================
TROUBLESHOOTING
================================================================================

Backend won't start on Railway:
  - Check logs in Railway dashboard
  - Make sure DATABASE_URL starts with postgresql+asyncpg://
  - Make sure all required env vars are set

Mobile app can't connect to backend:
  - Check EXPO_PUBLIC_API_URL points to your Railway URL
  - Make sure CORS_ORIGINS includes your mobile scheme (rgdgc://)
  - Try: curl <backend URL>/health from your laptop

Admin dashboard shows "Network Error":
  - Check VITE_API_URL points to Railway backend URL with /api/v1
  - Check CORS_ORIGINS includes the admin dashboard domain

Login fails:
  - The seeded admin account is: admin@rgdgc.com / admin123
  - For production, register a new account and promote via SQL

Google OAuth not working:
  - Check all 3 client IDs are set (web, iOS, Android)
  - Check authorized origins include your domains
  - iOS: make sure bundle ID matches com.rgdgc.app

Disc QR scan doesn't work:
  - The QR points to disc.rgdgc.com/{code}
  - You need to set up that domain to redirect to your backend
  - Or update the QR URL in config to use your Railway domain
