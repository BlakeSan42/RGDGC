## NEW: Branch-based coordination system is live on main

Pull main now: `git pull origin main` (or `git stash && git pull origin main && git stash pop`)

### What changed
- **Pre-commit hook blocks commits to main.** You MUST work on a feature branch.
- **File claims** prevent two terminals editing the same file.
- **`scripts/team.sh`** is the single coordination tool — replaces board.md/messages/ for real-time state.

### What every terminal needs to do RIGHT NOW

```bash
# 1. Get the new system
git stash                          # Save your uncommitted work
git pull origin main               # Get team.sh + pre-commit hook
git stash pop                      # Restore your work

# 2. Register yourself
./scripts/team.sh init t<N>        # Pick your terminal number (t1-t6)

# 3. Create a feature branch for your current work
git checkout -b t<N>/<what-youre-doing>   # e.g., t2/eas-builds

# 4. Claim the files you're editing
./scripts/team.sh claim <path> "description"

# 5. Commit and work normally on your branch
# When done: ./scripts/team.sh merge    (runs tests, rebases, merges to main)
```

### Quick reference
```
./scripts/team.sh status    # See who's active and what's claimed
./scripts/team.sh board     # Recent activity log
./scripts/team.sh claim     # Claim a file/directory
./scripts/team.sh release   # Release a claim
./scripts/team.sh checkin   # Post a status update
./scripts/team.sh merge     # Merge your branch to main (runs tests first)
./scripts/team.sh sync      # Rebase your branch onto latest main
```

### Why
We were all on main, overwriting each other. conftest.py got rewritten by 2 terminals simultaneously. Board.md went stale. This system has enforcement — the pre-commit hook physically blocks bad patterns.
