## PostGIS + Geo API is LIVE

### Location Update
River Grove is in **Kingwood, TX** (Houston metro, Harris County) — NOT River Grove, IL. All geo data sources should target Harris County, TX. TNRIS (Texas Natural Resources Info System) is the go-to for Texas LIDAR/DEM.

### What was done:
- Docker image swapped to `postgis/postgis:15-3.4-alpine`
- PostGIS 3.4 enabled on both main and test databases
- GeoAlchemy2 + Shapely added to requirements.txt
- **Hole model** now has: `tee_position` (Point), `basket_position` (Point), `fairway_line` (LineString), `tee_elevation_ft`, `basket_elevation_ft`, `elevation_change_ft`, `elevation_profile` (JSON)
- **Course model** now has: `boundary` (Polygon)
- **New `CourseFeature` model**: flexible geometry for OB zones, mandos, drop zones, trees, paths, water — any feature type with JSON properties
- GiST spatial indexes on all geometry columns
- **3 new API endpoints**:
  - `GET /api/v1/geo/courses/{id}/geojson` — full FeatureCollection (tees, baskets, fairways, OB, trees, etc.)
  - `GET /api/v1/geo/courses/{id}/holes/{n}/elevation` — elevation profile
  - `GET /api/v1/geo/nearest-hole?lat=X&lng=Y` — auto-detect hole from GPS position
- Alembic migration `bb0a57fd0a1a` for all schema changes
- Fixed test conftest to enable PostGIS on test DB — 40/41 pass
