

P0-1: Backend → Railway Deploy

  Time: ~10 minutes

  # 1. Install Railway CLI (if not already)
  brew install railway

  # 2. Login
  railway login

  # 3. Create project + link
  cd backend
  railway init        # creates project, pick a name like "rgdgc-backend"
  railway link        # links this directory

  # 4. Add PostgreSQL + Redis
  railway add --plugin postgresql
  railway add --plugin redis

  # 5. Set environment variables
  # Railway auto-sets DATABASE_URL and REDIS_URL from the plugins above,
  # but the backend expects asyncpg format. Set these:
  railway variables set \
    DATABASE_URL='${{Postgres.DATABASE_URL}}' \
    REDIS_URL='${{Redis.REDIS_URL}}' \
    ENVIRONMENT=production \
    SECRET_KEY="$(python3 -c 'import secrets; print(secrets.token_hex(32))')" \
    JWT_SECRET="$(python3 -c 'import secrets; print(secrets.token_hex(32))')" \
    OWNER_KEY="$(python3 -c 'import secrets; print(secrets.token_hex(32))')" \
    CORS_ORIGINS="https://rgdgc.com,https://admin.rgdgc.com,https://rgdgc-admin.vercel.app"

  # ⚠️   SAVE the OWNER_KEY somewhere safe — you'll need it for admin operations

  # 6. Deploy
  railway up

  # 7. Get your URL
  railway domain        # generates a *.up.railway.app URL

  # 8. Verify
  curl https://YOUR-URL.up.railway.app/health
  # Should return: {"status":"healthy","service":"rgdgc-api"}

  # 9. Seed production database
  railway run python -m scripts.seed_data
  railway run python -m scripts.seed_geo
  railway run python -m scripts.seed_elevation --offline

  Optional env vars (add when you have them):
  railway variables set \
    GOOGLE_CLIENT_ID="your-google-client-id" \
    GOOGLE_CLIENT_SECRET="your-google-secret" \
    ANTHROPIC_API_KEY="your-key" \
    WEB3_PROVIDER_URL="https://sepolia.infura.io/v3/YOUR_KEY"

  After deploy, note your Railway URL — you need it for P0-2.

  P0-2: Mobile → EAS Production Build

  Time: ~15 minutes (plus build queue)

  # 1. Install EAS CLI
  npm install -g eas-cli

  # 2. Login to Expo
  cd mobile
  eas login          # use your Expo account (or create one at expo.dev)

  # 3. Initialize EAS for this project
  eas init           # interactive — creates project on expo.dev, sets projectId

  # 4. Update app.json with your project ID
  # eas init does this automatically, but verify:
  # "extra.eas.projectId" should be set in app.json

  # 5. Set the production API URL (use your Railway URL from P0-1)
  # Edit mobile/eas.json — the "production" profile:
  #   "env": { "EXPO_PUBLIC_API_URL": "https://YOUR-URL.up.railway.app" }

  # 6. Build for iOS
  eas build --platform ios --profile production
  # First time: prompts for Apple Developer account (apple ID + team ID)
  # If you don't have certs, EAS can manage them for you (recommended)

  # 7. Build for Android
  eas build --platform android --profile production
  # First time: EAS generates a keystore for you (recommended)

  # 8. Submit to stores (when ready)
  eas submit --platform ios      # needs ASC API key or Apple ID
  eas submit --platform android  # needs Google Play service account JSON

  # 9. For now, test with a preview build on your phone:
  eas build --platform ios --profile preview
  # Scan the QR code when build completes to install

  GitHub Actions secret (for CI auto-deploys):
  # Get your Expo token from expo.dev → Account Settings → Access Tokens
  # Then add it to GitHub:
  gh secret set EXPO_TOKEN --body "your-expo-token"

  P0-3: Clean Up ace-bot/ Directory

  Time: ~2 minutes

  The real bot is in-app (backend/app/services/chat_service.py). The ace-bot/ directory is legacy Telegram/Discord code that confuses every session. You have two
  options:

  Option A — Delete it entirely (recommended):
  rm -rf ace-bot/
  git add -A ace-bot/
  git commit -m "Remove legacy ace-bot/ — real bot is in-app via chat_service.py"

  Option B — Keep it as a standalone Telegram companion (if you want that too):
  # It works — just needs tokens. Deploy to Railway as a separate service:
  cd ace-bot
  railway init       # separate project from backend
  railway variables set \
    RGDGC_API_URL="https://YOUR-URL.up.railway.app/api/v1" \
    TELEGRAM_BOT_TOKEN="your-telegram-bot-token" \
    ANTHROPIC_API_KEY="your-anthropic-key"
  railway up
